Analytical approach

PhishTank Analytical Approach

Approach: Classification data mining approach techniques were utilized in distinguishing phishing versus non-phishing websites.

Sources: Data sources such as Phishtank, Google search engine and MillerSmilaes which contain lists of phishing URL sites, IP addresses, etc., are useful in running this kind of analysis.​

Insight: Classification could lead to derived insights such as targeted industries prone to attacks, enabling security analyst to realign their focus in these areas.

Value: Classification techniques using URL's could allow an organization to develop proactive cybersecurity measures such as URL site restrictions in addition to maintaining its internal phishing blacklist automatically.​

  • Classification Algorithm: Rules Decision table
  • Data consisted of 4 attributes and 2019 instances 
  • 95.8% of the data was accurately classified 
  • Data was classified into two buckets, "Yes" or "No"
  • Cross-validation utilized: 10 Fold
  • Top 3 Targeted Industries: Financial Services, Retail and Social Media

Feodo Tracker Analytical Approach

Approach: Utilizing Classification of IP Address data from Feodo Tracker data can enable companies to  predict if an IP address is Dridex, Qakbot and Trickbot.

Sources: Data sources that contain at least one discrete piece of data can be used in the process of classification.​

Insight: Classification could lead to derived insights such as identifying trends in the countries or regions that are utilizing financial specific malware vs. other types of malware.

Value: Utilizing Classification of IP Address data from Feodo Tracker data can enable companies to  predict if an IP address is Dridex, Qakbot and Trickbot.​

  • Classification Algorithm: J48 Decision Tree
  • 80.76% of the data was correctly classified 
  • Data was classified into three buckets: “a=Dridex”, “b=TrickBot”, “c=QakBot”
  • Cross-validation utilized: Percentage Split
  • Most common malware type in the dataset is QakBot.

Stratosphere Remote Access Trojan (RAT) Dataset Analytical Approach

Approach: Event Correlation data mining approach techniques were utilized in identifying patterns between regular usage and Malware affected events.

Sources: Device logs, application logs and host logs are gathered, converted to required format and structure and plotted on a graph to view event patterns.​

Insight: Based on the logs, Remote Access Trojans(RATs) gained remote control of the compromised device and performed malicious activities at specific events. Event Correlation technique looks for relationships in timing and sequence of events and helps in spotting the patterns.

Value: Event Correlation can provide data intelligence, operation support, root cause analysis, and fraud detection. Additional benefits are real-time threat visibility, constant monitoring of threats, and smarter responses to incidents.

  • Logs captured from network traffic in cell phones infected with real Android RATs.
  • Time log of all the actions performed in the device and server, such as taking a picture, reading files, etc.
  1. Time-based events needed to be correlated and analyzed.
  2. The log file was split into different columns by converting it to a .csv file based on delimiters.
  3. Actions were categorized as events and used to differentiate between real events and trojan events.
  4. Columns that were analyzed include Timestamp, Module, and Event.
  5. Event Id is assigned to each Event for the purpose of plotting visualizations.
